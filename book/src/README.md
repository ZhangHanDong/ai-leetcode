# AI LeetCode：从题目到算法模式

> AI 时代，寻回写代码的乐趣

这本书的灵感，来自今年的两趟旅行。我在东京和巴黎看了好几场展览：美术、科技、人文、雕塑、家具、历史，不一而足。

和以往不同的是，这一次，我让 AI 全程深度参与其中。结果，我发现了一个新世界。

举个例子。

去年在阿姆斯特丹，我也逛过梵高美术馆，当时同样借助了 AI，比如用 ChatGPT 问过几幅梵高的画，但那种深度沉浸的感觉并不强烈。

今年不一样了。我把每次观展拍下的照片，都系统地发给 AI（主要是 Claude）。它不仅能逐一解读展品与画作本身，更让我意外的是，还挖掘出了同一座美术馆里、不同展厅之间的主题呼应。那背后，其实是策展人的视角；它甚至把我带回展品诞生的那个年代，还原出那段历史的大背景，让我真正读懂、也真正欣赏了眼前的一切。那种深度，是过去从未有过的。

而最近，我忽然意识到，读代码竟也成了我的一种思维游戏（毕竟，我已经很久没有静下心来直接读代码了）。于是我想：如果让 AI 陪着我一起刷题，会不会也能挖掘出当年独自刷题时未曾参透的“终极奥义”，也就是，算法背后的那些模式？

> **定位**：这本书关注算法知识如何迁移，而不仅是如何通过某一道题。读者可以从建模路径顺序阅读，也可以通过模式和题目索引按需查阅。

一道题的价值不止在答案。它还包含：

- 什么约束决定了解法；
- 什么状态足以描述问题；
- 哪个不变量保证算法正确；
- 条件变化后，原算法在哪里失效；
- 相同模式如何出现在其他题目中；
- 代码实现如何影响真实性能。

本书中的发布内容需要经过测试、验证、基准测试、可视化和编辑审阅。页面上的代码与性能结论最终都应能够追溯到仓库中的证据文件。
